## 작업 내용

### MVP 구현 (Phase 1)
- FastAPI 백엔드: drop(임시 전송) + vault(영구 보관) 이중 저장 구조
- CLI 클라이언트: 표준 라이브러리만 사용 (login/upload/download/list/delete + vault 서브커맨드)
- Docker 설정 (Dockerfile + docker-compose.yml)
- 만료 파일 정리 스크립트 (cleanup.py, cron용)

### Phase 2 — 웹 UI
- 임시 전송 / 보관함 탭 분리
- vault 폴더 탐색기 (breadcrumb 네비게이션, 새 폴더, 업로드, 삭제)
- 만료 셀렉터를 드롭존 내부로 이동 (UX 개선)
- 파일 검색 (drop/vault 각각)
- 이미지/비디오 썸네일 미리보기

### VPS 배포 (DevOps Stage 2)
- SSH 키 등록: 데스크탑 `C:\Users\user\.ssh\id_ed25519` → VPS `dev@46.250.251.82`
- git clone → `/opt/apps/quickdrop/`
- Python venv + pip install
- 환경변수: `/opt/envs/quickdrop.env` (심링크 → `.env`)
- 데이터: `/opt/data/quickdrop/drop/`, `/opt/data/quickdrop/vault/`
- systemd 서비스: `quickdrop.service` (enabled, active)
- ufw: 8200/tcp 개방

## VPS 구조

```
/opt/apps/quickdrop/          ← git clone (코드)
  .env → /opt/envs/quickdrop.env  (심링크)
  .venv/                      ← Python 가상환경
/opt/data/quickdrop/
  drop/                       ← 임시 파일 + .meta/
  vault/                      ← 영구 보관 (폴더 구조)
/opt/envs/quickdrop.env       ← 환경변수 (chmod 600)
```

## 핵심 결정 & 맥락
- **systemd 서비스**: 수동 실행 대신 systemd로 등록 → 서버 재부팅 시 자동 시작, 크래시 시 자동 재시작
- **비밀번호는 .env 기본값 유지**: VPS 배포 후 변경 필요 (현재 changeme)
- **데스크탑 SSH 키 신규 등록**: 노트북 SSH로 VPS 접속 → 데스크탑 공개키 authorized_keys에 추가
- **핸드폰 로컬 접속 불가 원인**: 데스크탑(192.168.55.x)과 핸드폰(192.168.35.x)이 다른 서브넷 → VPS 배포로 해결

# 만료 파일 정리 (cron 등록 예정)
