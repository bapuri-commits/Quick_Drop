## 작업 내용

### VPS 비밀번호 변경

### cleanup.py cron 등록
- 매일 UTC 03:00 만료 파일 자동 정리
- crontab: `0 3 * * * cd /opt/apps/quickdrop/backend && /opt/apps/quickdrop/.venv/bin/python cleanup.py >> /opt/logs/quickdrop/cleanup.log 2>&1`
- 로그 디렉토리: `/opt/logs/quickdrop/`

### Stage 4 — nginx 리버스 프록시
- nginx 1.24.0 설치
- 도메인 기반 가상 호스트: `drop.syworkspace.cloud` → proxy to `127.0.0.1:8200`
- `client_max_body_size 500M`, `proxy_read/send_timeout 600s`
- ufw 8200/tcp 외부 차단 → nginx(80/443)만 통과
- `news.syworkspace.cloud` → News_Agent 정적 서빙 (`/opt/data/news-agent/web/`)
- `syworkspace.cloud` → `drop.syworkspace.cloud`로 301 리다이렉트
- catch-all: 알 수 없는 호스트 → 444 (연결 끊기)

### Stage 5 — 도메인 & HTTPS
- 도메인: `syworkspace.cloud` (가비아 구매)
- DNS A 레코드: `@`, `drop`, `news` → `46.250.251.82` (TTL 600)
- certbot 2.9.0 + Let's Encrypt 인증서 발급 (3개 도메인 SAN 인증서)
- 인증서 만료: 2026-06-06 (자동 갱신 설정됨)
- HTTP → HTTPS 자동 리다이렉트 (certbot 자동 설정)

## 운영 명령어

```bash
sudo systemctl status/restart quickdrop
sudo systemctl status/restart/reload nginx
sudo journalctl -u quickdrop -f
sudo nginx -t
sudo certbot certificates
cd /opt/apps/quickdrop && git pull && sudo systemctl restart quickdrop
```
