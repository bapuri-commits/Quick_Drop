# QuickDrop Handoff — 2026-03-12

## 현재 상태
systemd에서 Docker 컨테이너로 전환 완료. VPS에서 정상 운영 중.

## 작업 내용
- `docker/docker-compose.yml` 보완: 바인드 마운트 3개 (drop, vault, clipboard), 환경변수 분리
- VPS에서 systemd 서비스 중지+disable → Docker 컨테이너로 교체
- `docker/.env` 생성 (VPS 전용, DATA_DIR + 시크릿)
- `.github/workflows/deploy.yml` 신규: Docker 기반 VPS 자동 배포
- cron cleanup을 `docker exec quickdrop python cleanup.py`로 전환

## 핵심 결정 & 맥락
- **바인드 마운트**: `/opt/data/quickdrop/{drop,vault,clipboard}` → 컨테이너 내 `/data/{drop,vault,clipboard}`. 기존 데이터 유실 없이 전환.
- **environment 우선**: compose의 `environment`가 `env_file`보다 우선하므로, 디렉토리 경로는 compose에서 고정, 시크릿만 `.env`에서 주입.
- **restart policy**: `unless-stopped` — 서버 재부팅 시 자동 시작.
