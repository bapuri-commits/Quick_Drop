# QuickDrop Handoff — 2026-03-08

## 현재 상태
클립보드 기능 구현 완료 (텍스트 + 이미지). 웹 UI 3탭 구조 + CLI clip 서브커맨드 추가. VPS 배포 후 재시작 필요.

## 작업 내용

### 클립보드 기능 (Phase 3)
- **백엔드 API**: `/api/clipboard` CRUD 엔드포인트 7개 (list, create, get, update, delete, delete-all, image-download)
- **저장소**: `backend/clipboard/` + `.meta/` (Drop과 동일한 파일 시스템 + JSON 메타데이터 패턴)
- **만료 옵션**: 1시간, 6시간, 1일, 7일, 영구 (초 단위로 전송, 0 = 영구)
- **텍스트**: JSON 메타데이터 내 인라인 저장 (최대 1MB)
- **이미지**: 별도 파일 저장 (최대 10MB), 전체 클립보드 용량 1GB 제한
- **웹 UI**: 3탭 구조 (임시 전송 / 보관함 / 클립보드), 모달 기반 추가/조회/편집
  - 텍스트: textarea 입력, 모달에서 내용 확인/수정/복사
  - 이미지: 파일 선택 또는 Ctrl+V 붙여넣기, 미리보기, clipboard API 복사
  - 검색, 전체 삭제, 토스트 알림
- **CLI**: `quickdrop clip {list|add|get|delete}` 서브커맨드
  - `clip add <title> <content>` 또는 stdin 파이프, `-f` 이미지 업로드
  - `-e 1h|6h|1d|7d|permanent` 만료 옵션
- **Cleanup**: cleanup.py에 클립보드 만료 정리 로직 추가 (expire_at null은 건너뜀)

### 변경 파일
- `backend/config.py` — CLIPBOARD_DIR, 용량 제한 설정
- `backend/main.py` — 클립보드 API 엔드포인트
- `backend/cleanup.py` — 클립보드 만료 정리
- `frontend/index.html` — 클립보드 탭/패널/모달 CSS+HTML+JS
- `cli/quickdrop.py` — clip 서브커맨드

## 핵심 결정 & 맥락
- **Drop과 동일한 저장 패턴**: DB 없이 파일 시스템 + JSON 메타데이터. 일관성 유지
- **텍스트는 메타데이터 내 인라인**: 별도 파일 불필요, 1MB 제한으로 충분
- **만료 옵션 단축형**: 클립보드 용도에 맞게 시간 단위 (1h, 6h) 추가, 영구 옵션도 지원
- **클립보드 accent color**: 초록/teal (#059669 light / #34d399 dark), Drop(파랑)·Vault(보라)와 구분
